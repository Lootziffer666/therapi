# THERAPI Design System (Stitch-ready, v0) Ziel: Google Stitch trotz eingeschränkter Befehlsgenauigkeit zuverlässig nutzen. ## 1) Produktkontext Desktop-first Web-Interface für ein lokales API-Discovery-Tool namens THERAPI. Fokus: HTTP-Capture, Endpoint-Übersicht, Schema-Versionen, Drift-Diff, OpenAPI-Export. Tooling-UI mit hoher Informationsdichte. ## 2) Design-Tokens - **Theme:** Ruhig, klar, produktiv. - **Radius:** 10px - **Grid/Spacing:** 8pt-Grid (4/8/16/24/32) - **Font:** Inter (Semibold headings, Regular body) ### Light-Tokens (Default) - primary: #1F6FEB - secondary: #5B7C99 - accent: #14B8A6 - success: #22C55E - warning: #F59E0B - danger: #EF4444 - info: #38BDF8 - background: #F8FAFC - surface: #FFFFFF - elevated-surface: #F1F5F9 - border: #CBD5E1 - text-primary: #0F172A - text-secondary: #475569 ### Dark-Tokens (Beleuchtung) - background: #0F172A - surface: #111827 - elevated-surface: #1F2937 - border: #334155 - text-primary: #E5E7EB - text-secondary: #94A3B8 ## 3) Seitenstruktur 1. Control Panel 2. Endpoint Explorer 3. Schema Drift 4. Capture Inspector 5. OpenAPI Export 6. Collections Export 7. Settings ## 4) Atomic Prompts (Siehe User-Input für Details zu A1, A2, C1-C4, E1-E2, D1, I1-I2, O1, X1, S1) ## 5) Qualitätsregeln - Stable shell preserved - Clear information hierarchy - Progressive disclosure applied - Low-noise visual density - Kind, actionable microcopy
